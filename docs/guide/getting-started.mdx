# Getting Started

<Warning>
  Breaking in <code>1.0.0</code>: if you are upgrading an older <code>simplex</code> install, run <code>openclaw simplex migrate</code> before switching commands and config to <code>openclaw-simplex</code>. See <a href="/guide/migration">Migration</a>.
</Warning>

Use this guide for the first working setup: install `simplex-chat`, run the WebSocket runtime, register the plugin, configure the channel, then pair the first contact.

<Steps>
  <Step title="Install SimpleX CLI">
    Official installer:

    ```bash
    curl -o- https://raw.githubusercontent.com/simplex-chat/simplex-chat/stable/install.sh | bash
    ```

    If the official installer resolves the wrong Darwin/Linux target on your host, use the temporary arch-matrix installer:

    ```bash
    curl -o- https://raw.githubusercontent.com/dangoldbj/simplex-chat/install-arch-matrix/install.sh | bash
    ```

    Verify the CLI:

    ```bash
    simplex-chat -h
    ```
  </Step>

  <Step title="Start the SimpleX runtime">
    Run the WebSocket runtime in a separate terminal:

    ```bash
    simplex-chat -p 5225
    ```

    <Info>
      This is a long-running foreground process. If you want it to start automatically, use the host-managed service examples in <a href="/guide/setup">Setup</a>.
    </Info>
  </Step>

  <Step title="Install and enable the plugin">
    Install the plugin in OpenClaw:

    ```bash
    openclaw plugins install @dangoldbj/openclaw-simplex
    openclaw plugins enable openclaw-simplex
    ```

    Trust it explicitly:

    ```bash
    openclaw config set plugins.allow "$(
      (openclaw config get plugins.allow --json 2>/dev/null || echo '[]') \
      | jq -c '. + ["openclaw-simplex"] | unique'
    )" --strict-json
    ```

    <Info>
      This release no longer needs the unsafe-install override because the plugin no longer starts <code>simplex-chat</code> itself.
    </Info>
  </Step>

  <Step title="Configure the channel">
    Quick CLI path:

    ```bash
    openclaw channels add --channel openclaw-simplex --url ws://127.0.0.1:5225
    ```

    Equivalent config:

    ```json
    {
      "channels": {
        "openclaw-simplex": {
          "enabled": true,
          "connection": {
            "wsUrl": "ws://127.0.0.1:5225"
          },
          "dmPolicy": "pairing",
          "allowFrom": ["*"]
        }
      }
    }
    ```

    <Info>
      OpenClaw will not start the channel until <code>channels.openclaw-simplex.connection</code> exists. If <code>simplex-chat</code> is not running at that endpoint, OpenClaw will mark the channel disconnected and record the connection error in channel status.
    </Info>
  </Step>

  <Step title="Verify the plugin and runtime">
    ```bash
    openclaw plugins list
    openclaw plugins info openclaw-simplex
    ```

    In Control UI, open <code>Control -&gt; Channels -&gt; SimpleX</code>.

    <Note>
      The current SimpleX card is a config editor. OpenClaw does not currently expose plugin-defined custom invite buttons on the channel card for this plugin.
    </Note>
  </Step>
</Steps>

## Create an Invite

The simplest user-facing path is the plugin CLI:

Create a one-time invite link:

```bash
openclaw simplex invite create --qr
```

For address links:

Show the current address link:

```bash
openclaw simplex address show --qr
```

Revoke the current address link:

```bash
openclaw simplex address revoke
```

You can still use the `simplex-chat` console directly:

```text
/c
/ad
/show_address
/delete_address
```

For automation, OpenClaw also exposes:

- `simplex.invite.create`
- `simplex.invite.list`
- `simplex.invite.revoke`

See [Gateway Methods](/reference/gateway-methods) for request and response shapes, including QR data URL fields.

## Pair the First Contact

1. Open the SimpleX app with the returned link.
2. Send a message to the OpenClaw contact.
3. List pairing requests:

```bash
openclaw pairing list
```

4. Approve the request:

```bash
openclaw pairing approve openclaw-simplex <code>
```

5. Send another message and verify the response.

For a visual walkthrough, see [Screenshots](/reference/screenshots).
