# Migration to `openclaw-simplex`

<Info>
  <code>1.0.0</code> is a breaking release. It renames both the plugin id and the channel id, and it removes managed mode.
</Info>

Use this guide if you previously configured or paired the plugin under the old `simplex` ids.

`1.0.0` renames both the plugin id and the channel id:

- plugin id: `simplex` -> `openclaw-simplex`
- channel id: `simplex` -> `openclaw-simplex`

It also removes managed mode. OpenClaw no longer starts `simplex-chat` for you; run it separately and connect over WebSocket.

Gateway method names remain unchanged in this release:

- `simplex.invite.create`
- `simplex.invite.list`
- `simplex.invite.revoke`

<Info>
  The migration helper updates both config keys and stored pairing/allowlist state so existing approvals can carry forward.
</Info>

<Steps>
  <Step title="Run the migration helper">
    ```bash
    openclaw simplex migrate
    ```

    Dry run:

    ```bash
    openclaw simplex migrate --dry-run
    ```
  </Step>

  <Step title="Update commands and config">
    Use the new ids in commands:

    ```bash
    openclaw plugins enable openclaw-simplex
    ```

    ```bash
    openclaw plugins info openclaw-simplex
    ```

    ```bash
    openclaw channels add --channel openclaw-simplex --url ws://127.0.0.1:5225
    ```

    ```bash
    openclaw pairing approve openclaw-simplex <code>
    ```
  </Step>

  <Step title="Run the external runtime">
    Configure the runtime under:

    ```json
    {
      "channels": {
        "openclaw-simplex": {
          "connection": {
            "wsUrl": "ws://127.0.0.1:5225"
          }
        }
      }
    }
    ```
  </Step>
</Steps>

## What the migration helper updates

- `plugins.entries.simplex` -> `plugins.entries.openclaw-simplex`
- `plugins.installs.simplex` -> `plugins.installs.openclaw-simplex`
- `plugins.allow` / `plugins.deny`
- `channels.simplex` -> `channels.openclaw-simplex`
- OpenClaw pairing and allowlist state files under the OpenClaw state directory
