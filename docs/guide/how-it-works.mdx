---
title: How It Works
description: Understand the OpenClaw-to-SimpleX request and response flow.
---

<Steps>
  <Step title="OpenClaw loads the plugin">
    OpenClaw registers the <code>openclaw-simplex</code> channel. For disabled or unconfigured channels, it can use the lightweight setup entry before loading the full runtime entry.
  </Step>

  <Step title="The runtime connects over WebSocket">
    OpenClaw only starts the channel after explicit <code>channels.openclaw-simplex.connection</code> config exists. The channel then connects to the separately running <code>simplex-chat</code> WebSocket API.
  </Step>

  <Step title="Inbound events are normalized">
    Incoming SimpleX events are parsed into OpenClaw context, including peer/group routing information, message bodies, media handling, and runtime status updates.
  </Step>

  <Step title="Policies are applied">
    OpenClaw evaluates DM policy, allowlists, group policy, pairing state, and command authorization before forwarding work to the agent.
  </Step>

  <Step title="Replies go back through SimpleX">
    Message actions, media sends, edits, deletes, reactions, and group operations are translated back into SimpleX commands and sent through the same WebSocket connection.
  </Step>
</Steps>
