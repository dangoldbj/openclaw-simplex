title: Troubleshooting
description: Common issues when connecting OpenClaw to SimpleX and how to resolve them.
---

<Note>
  When the runtime is down, OpenClaw will not start it for you. The channel stays disconnected until <code>simplex-chat</code> is running at the configured endpoint.
</Note>

## Plugin not listed

- Run `openclaw plugins list`
- Verify `plugins.allow` includes `openclaw-simplex`
- Verify package installed in the same runtime environment as OpenClaw
- Restart OpenClaw

## Channel fails to start

- Check `channels.openclaw-simplex.connection`
- Validate `connection` values (`wsUrl`, `wsHost`, `wsPort`)
- Verify `simplex-chat` is running and exposing the expected WebSocket endpoint
- If the runtime is down, OpenClaw will keep the channel disconnected and record the connection error in channel status
- For persistent startup, run `simplex-chat` under a host-managed service. See [Setup](/guide/setup).

## Inbound messages dropped

- Review `dmPolicy`, `allowFrom`, `groupPolicy`, `groupAllowFrom`
- In pairing mode, approve pending senders

## Invite methods failing

- Ensure channel account is configured and enabled
- Verify WS connectivity to SimpleX endpoint
- Check runtime status for last error

## Media issues

- Ensure media URLs are reachable by OpenClaw host
- Check configured max media size (`mediaMaxMb`)
- Confirm MIME/file handling on source side

## Useful diagnostic commands

```bash
simplex-chat -p 5225
```

```bash
openclaw plugins list
```

```bash
openclaw plugins info openclaw-simplex
```

```bash
openclaw channels add --channel openclaw-simplex --url ws://127.0.0.1:5225
```

If you need persistent runtime startup, see [Setup](/guide/setup).
