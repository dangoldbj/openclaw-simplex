# Security Model

This plugin enforces channel-side controls before handing requests to the agent.

<Warning>
  Start conservative: use <code>dmPolicy: "pairing"</code> and <code>groupPolicy: "allowlist"</code> until you have a reason to widen access.
</Warning>

## Access controls

<Columns cols={2}>
  <Card title="DM policy" icon="message-square">
    <code>open</code>, <code>allowlist</code>, <code>pairing</code>, or <code>disabled</code>.
  </Card>
  <Card title="Allowlist" icon="list-checks">
    Explicit sender allowlist through <code>allowFrom</code>.
  </Card>
</Columns>

<Columns cols={2}>
  <Card title="Group policy" icon="users">
    <code>open</code>, <code>allowlist</code>, or <code>disabled</code> for group traffic.
  </Card>
  <Card title="Pairing" icon="key-round">
    Explicit approval flow for unknown senders before granting access.
  </Card>
</Columns>

## Operational boundaries

- OpenClaw only gets WebSocket access to an already-running SimpleX endpoint.
- You decide where each process runs and what network path is allowed.

## Recommended production posture

- Start with `dmPolicy: "pairing"` and `groupPolicy: "allowlist"`.
- Keep `allowFrom`/`groupAllowFrom` narrow.
- External runtime mode is the supported deployment model for this plugin.
- Monitor status snapshots and logs for repeated authorization failures.
