---
title: Setup
description: Run simplex-chat as an external runtime and point OpenClaw at its WebSocket endpoint.
---

Prerequisite: install `simplex-chat` first (see [Getting Started](/guide/getting-started)).

<Info>
  OpenClaw does not supervise <code>simplex-chat</code> for external plugins. The supported model is: you run the runtime, OpenClaw connects to it over WebSocket.
</Info>

## Runtime

The plugin expects a reachable <code>simplex-chat</code> WebSocket endpoint. Start that runtime yourself, then wire OpenClaw to it through <code>channels.openclaw-simplex.connection.wsUrl</code>.

Start the runtime:

```bash
simplex-chat -p 5225
```

<Tip>
  For persistent startup, use one of the service options below.
</Tip>

```json
{
  "channels": {
    "openclaw-simplex": {
      "connection": {
        "wsUrl": "ws://127.0.0.1:5225"
      }
    }
  }
}
```

If the runtime is not available at that endpoint, OpenClaw keeps the channel disconnected and surfaces the connection error in channel status.

Manual invite and address management is still available in the <code>simplex-chat</code> console:

```text
/c
/ad
/show_address
/delete_address
```

## Run as a service

If you want the SimpleX runtime to come up automatically on login or machine boot, let the host OS supervise it and keep OpenClaw focused on the channel connection itself.

### Linux (`systemd --user`)

Create the user service:

```bash
mkdir -p ~/.config/systemd/user
cat > ~/.config/systemd/user/simplex-chat.service <<'EOF'
[Unit]
Description=SimpleX Chat WebSocket Runtime
After=network-online.target

[Service]
ExecStart=%h/.local/bin/simplex-chat -p 5225
Restart=on-failure
RestartSec=5

[Install]
WantedBy=default.target
EOF
```

Then enable and start it:

Reload user units:

```bash
systemctl --user daemon-reload
```

Enable and start the service:

```bash
systemctl --user enable --now simplex-chat.service
```

Check status:

```bash
systemctl --user status simplex-chat.service
```

<Note>
  Adjust the <code>ExecStart</code> path if <code>simplex-chat</code> is installed somewhere other than <code>%h/.local/bin/simplex-chat</code>.
</Note>

### macOS (`launchd`)

Create the launch agent:

```bash
mkdir -p ~/Library/LaunchAgents
cat > ~/Library/LaunchAgents/ai.openclaw.simplex-chat.plist <<'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
  <dict>
    <key>Label</key>
    <string>ai.openclaw.simplex-chat</string>
    <key>ProgramArguments</key>
    <array>
      <string>/usr/local/bin/simplex-chat</string>
      <string>-p</string>
      <string>5225</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
  </dict>
</plist>
EOF
```

Then load it:

Load the launch agent:

```bash
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/ai.openclaw.simplex-chat.plist
```

Restart it explicitly:

```bash
launchctl kickstart -k gui/$(id -u)/ai.openclaw.simplex-chat
```

Inspect status:

```bash
launchctl print gui/$(id -u)/ai.openclaw.simplex-chat
```

<Note>
  Adjust <code>/usr/local/bin/simplex-chat</code> if the binary is installed elsewhere on your system.
</Note>

## Per-account config

```json
{
  "channels": {
    "openclaw-simplex": {
      "accounts": {
        "ops": {
          "enabled": true,
          "connection": {
            "wsUrl": "ws://127.0.0.1:7777"
          }
        }
      }
    }
  }
}
```
