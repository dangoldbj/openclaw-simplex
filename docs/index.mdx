---
title: OpenClaw SimpleX
description: Invitation-first SimpleX messaging for OpenClaw, without phone numbers or hosted bot identities.
---

Your OpenClaw agent, reachable only by people you have invited: no phone number, no bot username, and no hosted bot platform account in the middle.

Within OpenClaw's channel ecosystem, this plugin enables a communication model that does not rely on platform identity or bot registration. With <a href="https://simplex.chat/">SimpleX</a>, the invite or address link becomes the access boundary: generate it, share it intentionally, revoke it when needed, and the agent does not expose a platform-discoverable bot identity.

<Info>
  Breaking in <code>1.0.0</code>: the plugin id and channel id are now <code>openclaw-simplex</code>, managed mode was removed, and existing users should run <code>openclaw simplex migrate</code>. See <a href="/guide/migration">Migration</a>.
</Info>

The implementation keeps developer-facing boundaries explicit: OpenClaw enforces policy, the plugin translates channel events and actions, and a separately managed <code>simplex-chat</code> runtime handles the SimpleX WebSocket API.

## Quick Start

Fresh install:

Install `simplex-chat`:

```bash
curl -o- https://raw.githubusercontent.com/simplex-chat/simplex-chat/stable/install.sh | bash
```

Run the SimpleX runtime in another terminal:

```bash
simplex-chat -p 5225
```

Install and enable the plugin:

```bash
openclaw plugins install @dangoldbj/openclaw-simplex
```

```bash
openclaw plugins enable openclaw-simplex
```

Configure the channel:

```bash
openclaw channels add --channel openclaw-simplex --url ws://127.0.0.1:5225
```

For the full setup flow, trust configuration, pairing, invite generation, and persistent runtime setup, see [Getting Started](/guide/getting-started) and [Setup](/guide/setup).

Upgrade from older `simplex` ids:

```bash
openclaw simplex migrate
```

See [Migration](/guide/migration) for the full breaking-change checklist.
